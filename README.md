# Telegram exchange bots

Два Telegram-бота для обмена крипты на рубли и обратно:

- `client_bot` принимает заявки от клиентов.
- `admin_bot` получает эти заявки, меняет статусы и управляет курсами.

Оба бота работают с общим файловым хранилищем, поэтому это удобный MVP для старта. Позже можно заменить JSON на PostgreSQL, добавить интеграции с блокчейном, платежками, AML/KYC и бухгалтерией.

## Что умеет MVP

- создание заявок на обмен `RUB -> crypto`
- создание заявок на обмен `crypto -> RUB`
- расчет суммы по текущим курсам
- отдельный бот для админов
- статусы заявки: `new`, `in_progress`, `completed`, `cancelled`
- уведомления клиенту о смене статуса заявки
- просмотр и изменение курсов через админ-бота

## Запуск

1. Установите зависимости:

```bash
pip install -r requirements.txt
```

2. Создайте `.env` по примеру `.env.example`.

3. Укажите:

- `CLIENT_BOT_TOKEN` - токен клиентского бота
- `ADMIN_BOT_TOKEN` - токен админ-бота
- `ADMIN_IDS` - Telegram ID админов через запятую

4. Запустите оба бота одной командой:

```bash
python -m exchange_bots
```

Если нужно, их все еще можно запускать отдельно:

```bash
python -m exchange_bots.client_bot
python -m exchange_bots.admin_bot
```

## Команды админ-бота

- `/start` - меню администратора
- `/pending` - новые и взятые в работу заявки
- `/orders` - последние 10 заявок
- `/rates` - показать курсы
- `/setrate USDT buy 98.5` - курс для `RUB -> USDT`
- `/setrate USDT sell 96.0` - курс для `USDT -> RUB`

## Важные замечания

Это техническая основа. Для реальной работы обменника вам почти наверняка понадобятся:

- юридическая проверка модели работы в вашей юрисдикции
- политика AML/KYC
- защита персональных данных
- аудит логики выплат и подтверждений оплаты
- журнал действий администраторов
- нормальная БД и резервное копирование
- защита от спама, флуда и поддельных чеков
